# RetriFlow PRD

## 产品概述

RetriFlow 是一个从零构建的企业级 Agentic RAG 系统，使用 Python + FastAPI + Vue 3 + TypeScript 实现。系统围绕企业知识问答场景，提供文档解析、结构化入库、智能分块、向量化、混合检索、问题重写、意图识别、MCP 工具调用、多轮记忆、流式回答、来源引用、链路追踪和后台运营管理能力。

首页默认就是对话入口，用户不需要先选择知识库。后端会根据会话上下文、意图树、知识库路由画像、关键词映射和实时工具能力自动完成问答链路编排。后台用于维护知识库、文档、分块、数据通道、意图树、关键词映射、示例问题、模型健康、用户、系统配置和 trace 诊断。

## 目标用户

- 需要搭建企业知识库问答系统的开发团队。
- 需要维护知识库、文档、分块、索引和检索效果的运营人员。
- 需要在 RAG 问答中接入 MCP 工具、会话记忆、链路追踪和后台配置的 AI 应用开发者。
- 需要学习企业级 RAG 工程落地链路的 GitHub 用户。

## 核心能力

### 登录与权限

- 支持注册、登录和 Bearer Token 鉴权。
- 支持普通用户和 admin 角色。
- 后台管理操作仅 admin 可用，普通用户访问后台时给出明确权限提示。

### 首页聊天

- 首页就是聊天入口，不要求用户先选择知识库。
- 支持 SSE 流式回答、Markdown 渲染、代码块、表格、列表和引用来源展示。
- 支持深度思考开关、智能联网开关、复制、重新生成、点赞和点踩反馈。
- 欢迎页示例问题与推荐问法由后台独立配置，不绑定具体知识库。
- 新会话首次提问后会以首个问题命名会话，并保持会话列表顺序稳定。

### 会话与记忆

- 支持会话创建、列表、删除、历史消息持久化。
- 支持短期摘要记忆、中期记忆和长期记忆。
- 记忆链路提供后台诊断接口，便于排查 Prompt 注入内容和上下文来源。

### 知识库管理

- 支持知识库新增、修改、列表、搜索和统计。
- 支持 embedding 模型、collection name、描述、示例问题、Prompt 配置和高级参数。
- collection name 仅允许小写英文字母和数字。
- 新增或修改时校验知识库名称和 collection name 重复，并在前端展示后端业务提示。
- 删除知识库时同步删除文档、分块、入库任务、向量记录和对应对象存储 bucket。
- 支持知识库路由画像、示例问题和关键词配置。

### 文档管理

- 支持上传、修改、列表、搜索、预览、重新索引和删除。
- 文档修改使用与上传一致的弹窗表单，并填入已有配置。
- 上传面板保留处理模式、切块策略、数据通道、chunk size、overlap 等配置。
- 上传源文件先进入可替换文件存储，支持本地 `local://` 与 RustFS/S3 `s3://bucket/key` URI。
- 同一知识库内不允许重复上传完全相同的源文件；后端按 SHA-256 判重，前端展示明确业务提示。
- 对象存储 key 使用内容 hash 前缀和原始中文文件名，避免随机乱码文件名。
- 支持 Tika 解析、结构化块、表格单元格、图片说明、OCR 降级和标准化清洗。

### 分块管理

- 支持固定大小、递归、语义、混合和结构感知分块。
- 默认策略为结构感知分块。
- 结构感知分块保留 Markdown 标题、段落、表格和代码块边界，默认 overlap 为 0。
- 分块详情支持分页，长内容按固定长度省略展示。

### 检索与答案生成

- 支持 BM25、向量召回、RRF 融合、rerank 和 final topK。
- 检索流程抽象为 SearchChannel 和 SearchResultPostProcessor。
- 意图树路由可保留多个阈值以上的 KB 或 MCP 候选，最多 3 个目标参与后续检索和工具调用。
- workflow metadata 暴露检索 stage counts、stage metrics、候选 KB、候选路径、置信度和 route topK。
- 聊天主链路通过 LangGraph `StateGraph` 编排，非流式请求覆盖记忆加载、上下文准备和答案生成，流式请求先完成上下文准备再由 SSE 承接生成。
- 使用场景化 Prompt 模板加载和渲染，覆盖 rewrite、intent、guidance、memory、context 和 answer 场景。
- 答案后处理包含引用补齐、来源展示、安全过滤、Markdown 格式整理和兜底回复。

### MCP 工具调用

- 支持内置工具和远程 MCP Server。
- 支持单轮多工具顺序或并行执行。
- 支持意图树 MCP 节点通过 `mcp_tool_id` 路由到具体工具，并支持节点级 `param_prompt_template` 参与工具参数提取。
- 远程 MCP Server 初始化失败时仅标记该 server 不健康并跳过，不影响内置工具和其他远程 server。
- 工具调用失败返回结构化错误，不拖垮整条问答链路。
- 工具执行接入节点级 trace，并记录 server、transport 和 schema version。

### 模型健康

- 支持 provider/model/capability 级健康快照、失败计数、healthy/open/half_open 三态熔断。
- 支持持久化恢复、主动探测接口、半开并发保护和 provider fallback。
- 后台提供模型健康 API、可视化面板、手动探测、启动预热和流式首包探针。

### 链路 Trace

- 支持 `rag_trace_nodes` 节点级 trace。
- 同步和流式聊天均记录真实运行节点。
- 后台 Trace 列表支持按 trace id、session id、task id、用户、状态和起止时间分页筛选。
- Trace 详情以执行耗时表展示完整 RAG 链路中每个环节的耗时、输入摘要、输出摘要和错误信息。
- 可选开启 LangSmith 外部 tracing，用于研发观测；后台 trace 仍是产品内链路诊断的事实来源。

### 队列限流

- 支持内存 FIFO 队列和 Redis 队列。
- Redis 使用 Docker 服务，基于 ZSET、entry TTL、Lua claim、active permit 和 release publish。
- SSE 会先发送 `event: queue`，内容来自 limiter snapshot。
- 超时拒绝会持久化用户问题和助手拒绝回复，并发送 `reject`、`final`、`done`。
- 等待中断开连接会清理 memory/Redis 等待项。

## 后台界面要求

- 后台保持左侧侧边栏布局，缩放和窄屏时侧边栏不能跳到顶部。
- 侧边栏每个入口都有清晰 icon。
- 知识库、文档、分块等列表统一显示分页与总数。
- 搜索框只在 focus 且有输入时显示下拉结果，点击结果后跳转。
- 下拉框使用统一产品化样式，避免浏览器原生选择框的突兀感。
- 表格内容需要在当前宽度内合理展示，操作按钮不换行。
- Trace 详情采用执行耗时表，不堆叠无关调试卡片。

## 产品边界

- 不新增没有真实后端支撑的空功能。
- Elasticsearch 仅在引入真实后端实现后接入；当前不添加空服务。
- 后台页面必须基于真实接口数据，不使用无意义假数据。
- 所有链路耗时必须来自真实 trace 节点，不用时间戳差值伪造。
